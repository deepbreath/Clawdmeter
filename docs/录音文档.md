# 录音文档

本文记录 Clawdmeter 录音功能的当前实现状态、已验证结论和后续排查线索，用于后续开发记忆。

## 当前状态

- 录音已验证正常。
- 录音入口：
  - `Key3 / IO18` 双击开始或停止录音。
  - 串口命令 `rec` 开始或停止录音。
- 录音文件保存到 SD 卡：
  - WAV: `/recordings/pending/rec_<uptime>.wav`
  - metadata: `/recordings/pending/rec_<uptime>.json`
- 录音完成后可用串口命令 `recplay` 播放最近一次录音。
- 可用串口命令 `recanalyze` 分析最近一次录音文件的波形统计。

## 硬件与音频链路

- 麦克风 ADC: `ES7210`
- ES7210 I2C 地址: `0x40`
- ES7210 I2S:
  - MCLK: `GPIO42`
  - BCLK: `GPIO9`
  - LRCK / WS: `GPIO45`
  - DIN: `GPIO10`
- 扬声器 codec: `ES8311`
- ES8311 和 ES7210 共用 I2S 时钟相关引脚，录音和播放不能同时占用音频链路。

## 关键修复记录

### 录音文件无声

根因：ES7210 的有效音频数据不一定落在原先固定的 `mix` 槽位里，固定槽位会生成正常 WAV 文件，但 PCM 可能接近静音。

修复：

- 默认录音模式改为 `auto`。
- 录音启动 warmup 阶段读取真实 I2S RX 样本。
- 对 `slot0`、`slot1`、`slot2`、`slot3`、`swap0`、`swap1`、`swap2`、`swap3`、`mix` 做统计。
- 自动选择 AC RMS / peak 合理的输入模式。
- 串口会输出类似：

```text
recorder: auto selected slotX (peak=..., rms=..., ac_rms=..., avg_abs=..., signal=1)
```

后续如果再次出现静音，优先看这行日志和 metadata 里的 `capture_mode_selected`、`signal_ok`、`slot_peaks`。

### 录完播放无声

根因：`sd_play_wav()` 先设置 `s_wav_busy = true`，但 WAV 播放任务里调用的音频占用检查又把 `sd_wav_busy()` 算作 busy，导致 SD WAV 播放被自己挡住。

修复：

- 新增 `sound_try_start_sd_stream()`，SD WAV 播放时只检查 `s_playing` 和 `s_stream_active`，不把当前 SD WAV 自己算作冲突。

### Audio busy 后 Wi-Fi 没恢复

根因：录音开始前会先 `wifi_manager_pause(true)`，如果随后 `sound_try_claim_audio()` 失败，会直接返回，Wi-Fi 保持暂停。

修复：

- audio busy 分支补 `wifi_manager_pause(false)`。

## 串口命令

- `rec`: 开始/停止录音
- `recplay`: 播放最近一次录音
- `recanalyze`: 分析最近一次录音 WAV
- `recmode auto|slot0|slot1|slot2|slot3|swap0|swap1|swap2|swap3|mix|tdmraw|es8311`: 设置录音采集模式
- `recrate 16000|48000`: 设置采样率
- `sd`: 打印 SD 和 DMA 状态
- `mem`: 打印 heap / internal / DMA / PSRAM 状态
- `i2c`: 扫描 I2C 设备
- `wifi`: 打印 Wi-Fi 和上传状态
- `upload`: 请求上传 pending 录音

## 排查建议

如果 WAV 文件存在但没声音：

1. 看串口 `recorder: auto selected ... signal=1` 是否出现。
2. 看 metadata:
   - `capture_mode_selected`
   - `signal_ok`
   - `peak_abs`
   - `avg_abs`
   - `slot_peaks`
   - `waveform.rms`
3. 如果 `signal_ok=false` 或 `peak_abs` 很低，优先怀疑 ES7210 初始化、I2S RX 槽位、MCLK/BCLK/WS/DIN 引脚或麦克风硬件。
4. 如果 metadata 显示有信号但 `recplay` 无声，优先查播放链路和 SD WAV 播放占用判断。
5. 如果 `recplay` 正常但电脑播放无声，检查 WAV header 中 sample rate、channels、bits、data bytes 是否和实际 PCM 匹配。
